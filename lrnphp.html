📚 Introduction to PHP

Welcome to PHP Programming! This tutorial is designed specifically for beginners who want to learn web development. Don't worry if you've never programmed before - we'll start from the basics and build up your skills step by step.

🤔 What is PHP?

PHP stands for Hypertext Preprocessor. It is a powerful, widely-used open-source scripting language that runs on web servers. PHP is especially designed for creating dynamic and interactive websites.

💡 Understanding PHP in Simple Terms:

Think of PHP as the "brain" behind websites. While HTML creates the structure (like the skeleton) and CSS adds the style (like clothing), PHP adds the intelligence and interactivity. For example:

  • HTML creates a login form on your screen
  • CSS makes it look beautiful
  • PHP actually checks if your username and password are correct!

🌐 Why Learn PHP?

1. Easy to Learn

PHP has a simple and straightforward syntax that's perfect for beginners. If you know basic HTML, you're already halfway there!

2. Widely Used

Over 75% of websites use PHP! Popular platforms like WordPress, Facebook, and Wikipedia are built with PHP.

3. Great Career Opportunities

PHP developers are in high demand. Learning PHP opens doors to freelancing and full-time jobs.

4. Free and Open Source

PHP is completely free to use. You don't need to pay for licenses or expensive software.

⚙️ How Does PHP Work?

Understanding how PHP works will help you grasp the bigger picture. Here's the step-by-step process:

1

User Requests a Page

When someone types a website address (URL) in their browser, they're requesting a PHP file from the server.

2

Server Processes PHP

The web server finds the PHP file and executes all the PHP code inside it. This happens on the server, not on the user's computer.

3

PHP Generates HTML

PHP processes the code and generates pure HTML output based on the instructions you wrote.

4

Browser Displays Result

The server sends the HTML to the user's browser, which displays the final webpage. The user never sees your PHP code - only the result!

🔒 Important Security Note: Because PHP runs on the server, your code remains secure. Users can see the HTML output but never your actual PHP code, which may contain sensitive information like database passwords.

📝 Your First PHP Program

Let's write your very first PHP program! Don't worry about understanding everything right now - we'll explain each part.

Basic PHP Syntax:

All PHP code must be written between <?php and ?> tags. These tags tell the server "Hey, this is PHP code - please execute it!"

<?php
  echo "Hello, World!";
?>

🔍 Code Explanation:

  • <?php - This opens the PHP code block
  • echo - This is a command that prints text on the webpage
  • "Hello, World!" - This is the text we want to display
  • ; - Every PHP statement must end with a semicolon (think of it as a period in a sentence)
  • ?> - This closes the PHP code block

Mixing PHP with HTML:

One of PHP's superpowers is that you can mix it with HTML. This allows you to create dynamic content within your web pages:

<!DOCTYPE html>
<html>
<head>
    <title>My First PHP Page</title>
</head>
<body>
    <h1>Welcome to PHP!</h1>
    <?php
        echo "<p>Today's date is: " . date("Y-m-d") . "</p>";
        echo "<p>Current time is: " . date("H:i:s") . "</p>";
    ?>
    <p>This is regular HTML text.</p>
</body>
</html>

🔍 What's Happening Here?

In this example, we're combining HTML with PHP. The HTML creates the page structure, while PHP dynamically generates the current date and time. Every time someone loads this page, they'll see the updated date and time!

💬 PHP Comments

Comments are notes you write in your code that PHP ignores. They're super helpful for explaining what your code does, especially when you come back to it later.

<?php
  // This is a single-line comment
  echo "Hello!"; // You can also add comments after code
  
  /* This is a multi-line comment

     Great for longer explanations */
  
  # This is also a single-line comment (alternative style)
?>
💡 Pro Tip: Always write comments! They help you remember what your code does and make it easier for others to understand your work. Think of comments as leaving notes for your future self.

📦 PHP Variables

Variables are one of the most fundamental concepts in programming. Think of them as labeled containers or boxes where you can store information that your program needs to remember and use.

🤔 What is a Variable?

A variable is like a container with a name label on it. You can put different types of information inside this container (like numbers, text, or true/false values), and you can use that information throughout your program.

🌍 Real-World Analogy:

Imagine you're organizing your school supplies:

  • You have a pencil box labeled "pencils" containing pencils
  • A folder labeled "homework" containing your homework papers
  • A notebook labeled "math" for math notes

In PHP, variables work the same way - they're labeled containers holding different types of data!

📝 How to Create Variables in PHP

In PHP, all variable names start with a dollar sign $ followed by the variable name. This is what makes PHP unique compared to other languages!

<?php
  $name = "John";        // Storing text (string)
  $age = 25;             // Storing a number (integer)
  $price = 99.99;        // Storing a decimal number (float)
  $isStudent = true;     // Storing true/false (boolean)
?>

🔍 Understanding the Syntax:

  • $ - Always starts a variable name in PHP
  • name - The variable's name (you choose this)
  • = - The assignment operator (means "store this value")
  • "John" - The value you're storing
  • ; - Ends the statement

📋 Variable Naming Rules

When creating variables, you must follow these important rules:

✅ Correct Variable Names:

$name
$firstName
$age_in_years
$_total
$price2024

❌ Incorrect Variable Names:

$2name         // Can't start with a number
$first-name    // Can't use hyphens
$first name    // Can't have spaces
$@price        // Can't use special symbols (except _)
⚠️ Case Sensitivity: PHP variables are case-sensitive! This means $name, $Name, and $NAME are three completely different variables. Be consistent with your naming!

🎨 Data Types in PHP

Variables can store different types of data. PHP automatically detects the data type, which makes it beginner-friendly. Here are the main types:

1. String (Text)

Strings are sequences of characters - basically any text. Always enclosed in quotes.

<?php
  $greeting = "Hello, World!";
  $name = 'Alice';  // Single or double quotes both work
  $message = "Welcome to PHP programming!";
  
  echo $greeting;  // Outputs: Hello, World!
?>

2. Integer (Whole Numbers)

Integers are whole numbers without decimal points. They can be positive or negative.

<?php
  $age = 25;
  $year = 2024;
  $temperature = -10;
  $score = 0;
  
  echo $age;  // Outputs: 25
?>

3. Float (Decimal Numbers)

Floats are numbers with decimal points, also called "floating-point numbers."

<?php
  $price = 99.99;
  $weight = 65.5;
  $pi = 3.14159;
  
  echo $price;  // Outputs: 99.99
?>

4. Boolean (True/False)

Booleans represent yes/no, on/off, or true/false values. Very useful for conditions and logic.

<?php
  $isLoggedIn = true;
  $hasPermission = false;
  $isStudent = true;
  
  if ($isLoggedIn) {
    echo "Welcome back!";
  }
?>

🔄 Using Variables

Once you've stored data in a variable, you can use it anywhere in your code. This is where variables become really powerful!

<?php
  // Store user information
  $userName = "Sarah";
  $userAge = 22;
  $userCity = "New York";
  
  // Use the variables to create personalized messages
  echo "Hello, " . $userName . "!";
  echo "You are " . $userAge . " years old.";
  echo "You live in " . $userCity . ".";
  
  // Output:
  // Hello, Sarah!
  // You are 22 years old.
  // You live in New York.
?>

💡 Why Use Variables?

Imagine you need to use someone's name 50 times in your code. Without variables, you'd have to type "Sarah" 50 times. If you need to change it to "Sara" (different spelling), you'd have to find and change it 50 times!

With variables, you just change it once: $userName = "Sara"; and it updates everywhere!

🔢 Variable Operations

You can perform operations with variables just like you do with regular numbers or text:

Math Operations:

<?php
  $a = 10;
  $b = 5;
  
  $sum = $a + $b;        // Addition: 15
  $difference = $a - $b;  // Subtraction: 5
  $product = $a * $b;     // Multiplication: 50
  $quotient = $a / $b;    // Division: 2
  
  echo "Sum: " . $sum;
?>

String Operations (Concatenation):

<?php
  $firstName = "John";
  $lastName = "Doe";
  
  // Combining strings with a dot (.)
  $fullName = $firstName . " " . $lastName;
  echo $fullName;  // Outputs: John Doe
  
  // Alternative way (for double quotes only)
  echo "Hello, $firstName $lastName!";
?>

🌍 Variable Scope

Scope refers to where in your code a variable can be accessed. Think of it like different rooms in a house - some things are available everywhere, others only in specific rooms.

Global Scope (Available Everywhere):

<?php
  $globalName = "Alice";  // Declared outside any function
  
  function greet() {
    global $globalName;  // Must use 'global' keyword inside function
    echo "Hello, " . $globalName;
  }
  
  greet();  // Outputs: Hello, Alice
?>

Local Scope (Available Only Inside Function):

<?php
  function calculate() {
    $result = 10 + 5;  // Local variable
    echo $result;
  }
  
  calculate();  // Outputs: 15
  // echo $result;  // Error! $result doesn't exist outside the function
?>
💡 Best Practice: Use local variables whenever possible. They keep your code organized and prevent accidental changes to important data. Global variables should be used sparingly!

🔄 Changing Variable Values

Variables are called "variables" because their values can vary (change) during program execution:

<?php
  $score = 0;           // Starting score
  echo $score;          // Outputs: 0
  
  $score = 10;          // Score changed
  echo $score;          // Outputs: 10
  
  $score = $score + 5;  // Adding to current value
  echo $score;          // Outputs: 15
  
  // Shortcut for adding to current value:
  $score += 5;          // Same as $score = $score + 5;
  echo $score;          // Outputs: 20
?>

⚙️ PHP Functions

Functions are reusable blocks of code that perform specific tasks. They're like mini-programs within your main program. Once you create a function, you can use it over and over again without rewriting the same code!

🤔 What is a Function?

A function is a named block of code that does a specific job. Instead of writing the same code multiple times, you write it once inside a function and then "call" (use) that function whenever you need it.

🌍 Real-World Analogy:

Think of a function like a coffee machine:

  • Input: You put in coffee beans and water
  • Process: The machine grinds, brews, and heats
  • Output: You get a cup of coffee

You don't need to know how the machine works internally - you just press a button and get your coffee. Functions work the same way in programming!

✨ Why Use Functions?

1. Reusability

Write code once, use it many times. No need to copy-paste the same code everywhere.

2. Organization

Break down complex programs into smaller, manageable pieces. Your code becomes easier to read and understand.

3. Easier Debugging

If something goes wrong, you only need to fix it in one place - inside the function.

4. Teamwork

Different team members can work on different functions, making collaboration easier.

📝 Creating Your First Function

The basic syntax for creating a function in PHP is simple:

<?php
  function functionName() {
    // Code to be executed goes here
  }
?>

Simple Example:

<?php
  // Define the function
  function sayHello() {
    echo "Hello, welcome to PHP!";
  }
  
  // Call (use) the function
  sayHello();  // Outputs: Hello, welcome to PHP!
  
  // You can call it multiple times
  sayHello();  // Outputs: Hello, welcome to PHP!
  sayHello();  // Outputs: Hello, welcome to PHP!
?>

🔍 Understanding the Syntax:

  • function - Keyword that tells PHP you're creating a function
  • sayHello - The name you give to your function (you choose this)
  • () - Parentheses where parameters go (we'll learn about these next)
  • { } - Curly braces contain the code that runs when function is called

📥 Functions with Parameters

Parameters (also called arguments) are values you pass into a function to customize its behavior. They make functions much more flexible and powerful!

<?php
  // Function with one parameter
  function greetUser($name) {
    echo "Hello, " . $name . "! Welcome to our website.";
  }
  
  // Call the function with different names
  greetUser("Sarah");   // Outputs: Hello, Sarah! Welcome to our website.
  greetUser("John");    // Outputs: Hello, John! Welcome to our website.
  greetUser("Ahmed");   // Outputs: Hello, Ahmed! Welcome to our website.
?>

💡 Think of Parameters Like This:

Remember our coffee machine analogy? Parameters are like choosing different coffee types. The machine (function) stays the same, but you can make espresso, cappuccino, or latte by changing the input (parameter)!

Multiple Parameters:

Functions can accept multiple parameters separated by commas:

<?php
  function introduce($name, $age, $city) {
    echo "My name is " . $name . ".";
    echo "I am " . $age . " years old.";
    echo "I live in " . $city . ".";
  }
  
  introduce("Alice", 25, "New York");
  // Outputs:
  // My name is Alice.
  // I am 25 years old.
  // I live in New York.
?>

↩️ Functions with Return Values

Instead of just displaying information, functions can calculate something and return the result so you can use it elsewhere in your code.

<?php
  function add($a, $b) {
    $sum = $a + $b;
    return $sum;  // Send the result back
  }
  
  // Store the returned value in a variable
  $result = add(10, 5);
  echo $result;  // Outputs: 15
  
  // Or use it directly
  echo add(20, 30);  // Outputs: 50
  
  // Use in calculations
  $total = add(100, 50) + add(25, 75);
  echo $total;  // Outputs: 250 (150 + 100)
?>
🎯 Key Concept: When a function reaches a return statement, it immediately stops executing and sends the value back. Any code after return won't run!

🎨 Default Parameter Values

You can set default values for parameters. If someone calls the function without providing that parameter, the default value is used:

<?php
  function greet($name = "Guest", $time = "day") {
    echo "Good " . $time . ", " . $name . "!";
  }
  
  greet("Sarah", "morning");  // Outputs: Good morning, Sarah!
  greet("John");              // Outputs: Good day, John!
  greet();                    // Outputs: Good day, Guest!
?>
💡 Pro Tip: Default parameters should always come at the end of the parameter list. This prevents confusion when calling the function.

📝 PHP Strings

Strings are one of the most commonly used data types in programming. Almost every program you write will work with text data - whether it's user names, messages, email addresses, or product descriptions. Let's learn how to master strings in PHP!

🤔 What is a String?

A string is simply a sequence of characters - letters, numbers, symbols, or spaces. In programming terms, any text is a string. Your name is a string, this sentence is a string, even "12345" (in quotes) is a string!

🌍 Real-World Analogy:

Think of a string like a necklace or bracelet made of beads:

  • Each bead represents a character (letter, number, or symbol)
  • The beads are arranged in a specific order
  • You can add beads, remove beads, or rearrange them
  • The whole necklace is your string!

✍️ Creating Strings

In PHP, you can create strings using either single quotes ' or double quotes ":

<?php
  // Using double quotes
  $greeting = "Hello, World!";
  
  // Using single quotes
  $name = 'Sarah';
  
  // Strings can contain numbers
  $phoneNumber = "123-456-7890";
  
  // Empty string
  $empty = "";
  
  // Multi-line string (using double quotes)
  $message = "This is line one.
This is line two.
This is line three.";
?>

🔍 Single Quotes vs Double Quotes

There's an important difference between single and double quotes in PHP:

💡 Key Differences:

  • Double Quotes (""): PHP processes special characters and variables inside them
  • Single Quotes (''): PHP treats everything literally (exactly as written)
<?php
  $name = "John";
  
  // Double quotes - variable gets processed
  echo "Hello, $name!";     // Outputs: Hello, John!
  
  // Single quotes - variable is treated as text
  echo 'Hello, $name!';     // Outputs: Hello, $name!
  
  // Special characters work in double quotes
  echo "Line 1\nLine 2";    // \n creates a new line
  
  // Special characters don't work in single quotes
  echo 'Line 1\nLine 2';    // Outputs: Line 1\nLine 2 (literally)
?>
💡 Best Practice: Use double quotes when you need to include variables or special characters. Use single quotes for plain text - it's slightly faster!

🔗 String Concatenation (Joining Strings)

Concatenation means joining two or more strings together. In PHP, we use the dot operator . to concatenate strings:

<?php
  $firstName = "John";
  $lastName = "Doe";
  
  // Concatenating with dot operator
  $fullName = $firstName . " " . $lastName;
  echo $fullName;  // Outputs: John Doe
  
  // Concatenating multiple strings
  $greeting = "Hello, " . $firstName . " " . $lastName . "!";
  echo $greeting;  // Outputs: Hello, John Doe!
  
  // Concatenate and assign (shortcut)
  $message = "Welcome";
  $message .= " to PHP";  // Same as: $message = $message . " to PHP";
  echo $message;  // Outputs: Welcome to PHP
?>

📏 Useful String Functions

PHP provides many built-in functions to work with strings. Let's explore the most useful ones for beginners:

1. strlen() - Find String Length

Returns the number of characters in a string (including spaces):

<?php
  $text = "Hello World";
  $length = strlen($text);
  echo $length;  // Outputs: 11 (including the space)
  
  // Practical use: Validating input
  $password = "abc123";
  if (strlen($password) < 8) {
    echo "Password must be at least 8 characters!";
  }
?>

2. str_word_count() - Count Words

Counts the number of words in a string:

<?php
  $sentence = "PHP is fun to learn";
  $wordCount = str_word_count($sentence);
  echo $wordCount;  // Outputs: 5
  
  // Practical use: Blog post word counter
  $blogPost = "This is my first blog post about PHP programming.";
  echo "Word count: " . str_word_count($blogPost);
?>

3. strtoupper() and strtolower() - Change Case

Convert strings to uppercase or lowercase:

<?php
  $text = "Hello World";
  
  echo strtoupper($text);  // Outputs: HELLO WORLD
  echo strtolower($text);  // Outputs: hello world
  
  // Practical use: Comparing user input (case-insensitive)
  $userInput = "YES";
  if (strtolower($userInput) == "yes") {
    echo "User agreed!";
  }
?>

4. ucfirst() and ucwords() - Capitalize

Capitalize the first letter of a string or each word:

<?php
  $name = "john doe";
  
  echo ucfirst($name);   // Outputs: John doe
  echo ucwords($name);   // Outputs: John Doe
  
  // Practical use: Formatting names properly
  $userName = "sarah johnson";
  $formattedName = ucwords($userName);
  echo "Welcome, " . $formattedName . "!";
?>

5. trim() - Remove Extra Spaces

Removes whitespace from the beginning and end of a string:

<?php
  $text = "   Hello World   ";
  $cleaned = trim($text);
  echo $cleaned;  // Outputs: Hello World (no extra spaces)
  
  // Practical use: Cleaning user input
  $email = "  user@example.com  ";
  $cleanEmail = trim($email);
  echo $cleanEmail;  // Outputs: user@example.com
?>

6. substr() - Extract Part of String

Extracts a portion of a string:

<?php
  $text = "Hello World";
  
  // Extract from position 0, length 5
  echo substr($text, 0, 5);   // Outputs: Hello
  
  // Extract from position 6 to end
  echo substr($text, 6);      // Outputs: World
  
  // Extract last 5 characters
  echo substr($text, -5);     // Outputs: World
  
  // Practical use: Creating username from email
  $email = "john.doe@example.com";
  $username = substr($email, 0, strpos($email, "@"));
  echo $username;  // Outputs: john.doe
?>

7. str_replace() - Replace Text

Replaces all occurrences of a search string with a replacement:

<?php
  $text = "I love Java programming";
  
  // Replace Java with PHP
  $newText = str_replace("Java", "PHP", $text);
  echo $newText;  // Outputs: I love PHP programming
  
  // Replace multiple words
  $message = "Hello World World";
  $clean = str_replace("World", "PHP", $message);
  echo $clean;  // Outputs: Hello PHP PHP
  
  // Practical use: Censoring bad words
  $comment = "This is a bad word";
  $censored = str_replace("bad", "***", $comment);
  echo $censored;
?>

8. strpos() - Find Position

Finds the position of the first occurrence of a substring:

<?php
  $email = "user@example.com";
  
  // Find position of @
  $position = strpos($email, "@");

📊 PHP Arrays

Imagine you need to store information about 100 students. Would you create 100 separate variables? Of course not! That's where arrays come in. Arrays let you store multiple values in a single variable, making your code cleaner and more efficient.

🤔 What is an Array?

An array is a special variable that can hold multiple values at once. Instead of creating separate variables for related data, you can group them together in one array.

🌍 Real-World Analogy:

Think of an array like a filing cabinet:

  • The cabinet itself is the array variable
  • Each drawer is numbered (index 0, 1, 2, etc.)
  • Each drawer contains different documents (values)
  • You can quickly access any drawer by its number

Or think of it like a shopping list where each item has a position number!

📝 Types of Arrays in PHP

PHP has three types of arrays:

1. Indexed Arrays

Arrays with numeric index (0, 1, 2, 3...)

2. Associative Arrays

Arrays with named keys (like "name", "age", "email")

3. Multidimensional Arrays

Arrays containing other arrays

🔢 Indexed Arrays

Indexed arrays use numbers (starting from 0) to access elements:

<?php
  // Method 1: Using array() function
  $fruits = array("Apple", "Banana", "Orange");
  
  // Method 2: Using short syntax (modern way)
  $colors = ["Red", "Green", "Blue"];
  
  // Accessing array elements (remember: counting starts at 0!)
  echo $fruits[0];  // Outputs: Apple
  echo $fruits[1];  // Outputs: Banana
  echo $fruits[2];  // Outputs: Orange
  
  echo $colors[0];  // Outputs: Red
?>
⚠️ Remember: Array indexing starts at 0, not 1! So the first element is [0], the second is [1], and so on. This confuses beginners, but you'll get used to it quickly!

Creating and Modifying Indexed Arrays:

<?php
  // Create empty array
  $numbers = [];
  
  // Add elements
  $numbers[0] = 10;
  $numbers[1] = 20;
  $numbers[2] = 30;
  
  // Or add without specifying index (automatically uses next number)
  $numbers[] = 40;  // Becomes index 3
  $numbers[] = 50;  // Becomes index 4
  
  // Modify existing element
  $numbers[1] = 25;  // Change 20 to 25
  
  // Display array
  print_r($numbers);
?>

🏷️ Associative Arrays

Associative arrays use descriptive names (keys) instead of numbers. This makes your code more readable and meaningful:

<?php

  echo $student["age"];    // Outputs: 20
  echo $user["email"];     // Outputs: john@example.com
?>

Use associative arrays when you want to give meaningful names to your data. For example:

Visible: Data appears in the URL

Use for: Search queries, filters, non-sensitive data

Example: Google searches

POST Method

Hidden: Data sent in the background

Use for: Login forms, registration, sensitive data

Example: Login pages, sign-up forms

🛡️ Preventing XSS Attacks

XSS (Cross-Site Scripting) is when attackers inject malicious code into your website. Always sanitize user input:

<?php

    $data = stripslashes($data);        // Remove backslashes

    return $data;
  }
  
  if ($_SERVER["REQUEST_METHOD"] == "POST") {
    // Clean all input data
    $name = sanitize_input($_POST['name']);
    $email = sanitize_input($_POST['email']);

    echo "Email: " . $email . "<br>";
    echo "Message: " . $message;
  }
?>
🔐 Security Best Practices:
  • Always use htmlspecialchars() before displaying user input
  • Validate data on the server-side (not just with HTML5 validation)
  • Never trust user input - always sanitize it
  • Use prepared statements for database queries (we'll learn this in Database section)

🛡️ Secure Insert with Prepared Statements

<?php
  $conn = mysqli_connect("localhost", "root", "", "myWebsite");

?>

🍪 PHP Cookies

Have you ever noticed that websites remember your preferences, like your username or language choice, even after you close your browser? That's cookies at work! Cookies are small pieces of data stored on the user's computer that help websites remember information between visits.

🤔 What is a Cookie?

A cookie is a small text file (usually 4KB or less) that websites store on your computer. When you visit the website again, it reads the cookie to remember information about you.

🌍 Real-World Analogy:

Think of a cookie like a loyalty card at a coffee shop:

  • First visit: The shop gives you a loyalty card (cookie)
  • The card stores: Your name, favorite drink, points earned
  • Next visit: You show the card and they remember you
  • Expiration: Card might expire after 1 year

Cookies work exactly the same way on websites!

🎯 What are Cookies Used For?

Remembering Login

"Keep me logged in" checkbox uses cookies

Shopping Carts

Remembering items you added even after closing browser

User Preferences

Language, theme, font size choices

Tracking

Remembering which pages you visited

🍪 Creating a Cookie

In PHP, we use the setcookie() function to create cookies:

<?php
  // Basic syntax
  setcookie("username", "JohnDoe", time() + (86400 * 30), "/");
  
  // Breakdown:
  // "username" - Cookie name
  // "JohnDoe" - Cookie value
  // time() + (86400 * 30) - Expires in 30 days (86400 seconds = 1 day)
  // "/" - Available throughout entire website
  
  echo "Cookie has been set!";
?>
⚠️ Important: setcookie() must be called BEFORE any HTML output. It should be at the very top of your PHP file, before any <html> tags or echo statements.

📝 Complete Cookie Syntax

<?php
  setcookie(
    "cookieName",           // Name
    "cookieValue",          // Value
    time() + 3600,          // Expiration time (1 hour from now)
    "/",                    // Path (/ means entire website)
    "example.com",          // Domain
    true,                   // Secure (only HTTPS)
    true                    // HTTP only (not accessible via JavaScript)
  );
?>

?>

🔐 PHP Sessions

Sessions are like your personal locker at a gym - they store your information securely on the server while you use the website. Unlike cookies (stored on user's computer), sessions keep data on the server, making them more secure for sensitive information like login status and user preferences.

🤔 What is a Session?

A session is a way to store information temporarily on the server for a specific user. Each user gets a unique session ID, which acts like a key to their personal storage space on the server.

🌍 Real-World Analogy:

Think of sessions like a coat check at a restaurant:

  • You arrive: They give you a numbered ticket (session ID)
  • Your coat: Stored safely in the back (session data on server)
  • During visit: You keep the ticket to prove it's yours
  • You leave: Show ticket, get your coat back (session ends)
  • Lost ticket: Can't claim the coat (session expires)

🆚 Sessions vs Cookies

Sessions

  • Stored on server
  • More secure
  • Can store large data
  • Expires when browser closes
  • Best for sensitive data

Cookies

  • Stored on user's computer
  • Less secure
  • Limited to 4KB
  • Can last for years
  • Best for preferences

🚀 Starting a Session

Before you can use sessions, you must start one with session_start():

<?php
  // Always start session at the very beginning of your PHP file
  session_start();
  
  echo "Session started successfully!";
?>
⚠️ Critical Rule: session_start() MUST be called before any HTML output or echo statements. Put it at the very top of your PHP file!

💾 Storing Data in Sessions

Once session is started, use the $_SESSION superglobal array to store data:

<?php
  session_start();
  
  // Store single values
  $_SESSION["username"] = "JohnDoe";
  $_SESSION["email"] = "john@example.com";
  $_SESSION["user_id"] = 12345;
  $_SESSION["is_logged_in"] = true;
  
  echo "Session data stored successfully!";
?>

Storing Complex Data:

<?php
  session_start();
  
  // Store arrays
  $_SESSION["user_info"] = [
    "name" => "John Doe",
    "age" => 25,
    "role" => "admin"
  ];
  
  // Store shopping cart
  $_SESSION["cart"] = [
    ["product" => "Laptop", "price" => 599.99],
    ["product" => "Mouse", "price" => 29.99]
  ];

?>

⏰ Session Timeout Example

<?php
  session_start();

  $timeout_duration = 1800;
  
  // Check if session has last activity timestamp

    $elapsed_time = time() - $_SESSION["last_activity"];
    
    // If more than timeout duration has passed

      session_unset();

      exit();
    }
  }
  
  // Update last activity timestamp
  $_SESSION["last_activity"] = time();
?>

📁 PHP File System

Files are everywhere in web development - user uploads, configuration files, logs, data storage, and more. PHP provides powerful functions to create, read, write, and manage files on your server. Let's learn how to work with files safely and effectively!

🤔 What is File Handling?

File handling is the ability to create, read, write, update, and delete files on the server using PHP. This is essential for many web applications.

🌍 Real-World Uses:

  • User Uploads: Profile pictures, documents, videos
  • Data Storage: Saving user data, configurations
  • Logs: Recording errors, user activities
  • Content Management: Blog posts, articles
  • File Downloads: Serving PDFs, images, documents

📖 Reading Files

Method 1: file_get_contents() (Easiest)

Reads entire file into a string:

<?php
  // Read entire file
  $content = file_get_contents("data.txt");
  echo $content;
  
  // Check if file exists first
  if(file_exists("data.txt")) {
    $content = file_get_contents("data.txt");
    echo $content;
  } else {
    echo "File not found!";
  }
?>

Method 2: fopen(), fread(), fclose()

More control over file reading:

<?php
  // Open file for reading
  $file = fopen("data.txt", "r");
  
  if($file) {
    // Read entire file
    $content = fread($file, filesize("data.txt"));
    echo $content;
    
    // Always close the file
    fclose($file);
  } else {
    echo "Unable to open file";
  }
?>

Method 3: fgets() - Read Line by Line

<?php
  $file = fopen("data.txt", "r");
  
  if($file) {
    // Read file line by line
    while(!feof($file)) {
      $line = fgets($file);
      echo $line . "<br>";
    }
    
    fclose($file);
  }
?>

📝 Writing to Files

Method 1: file_put_contents() (Easiest)

<?php
  // Write to file (overwrites existing content)
  $data = "Hello, this is new content!";
  file_put_contents("data.txt", $data);
  
  echo "File written successfully!";
  
  // Append to file (adds to end without deleting existing content)
  $newData = "\nThis line is appended.";
  file_put_contents("data.txt", $newData, FILE_APPEND);
?>

Method 2: fopen(), fwrite(), fclose()

<?php
  // Open file for writing (creates if doesn't exist)
  $file = fopen("data.txt", "w");
  
  if($file) {
    fwrite($file, "First line of text\n");
    fwrite($file, "Second line of text\n");
    fwrite($file, "Third line of text\n");
    
    fclose($file);
    echo "File written successfully!";
  } else {
    echo "Unable to open file";
  }
?>

📋 File Opening Modes

Important Mode Flags:

  • r - Read only (file must exist)
  • r+ - Read and write (file must exist)
  • w - Write only (creates new or erases existing)
  • w+ - Read and write (creates new or erases existing)
  • a - Append (writes to end of file, creates if doesn't exist)
  • a+ - Read and append
  • x - Write only (creates new, fails if exists)

🔒 File Security Best Practices

Security Guidelines:

  • Validate file types: Check extensions and MIME types
  • Limit file sizes: Prevent large uploads that could fill your server
  • Rename uploaded files: Use unique names to prevent overwriting
  • Store outside web root: Don't allow direct access to uploaded files
  • Scan for viruses: Use antivirus tools for uploads
  • Set proper permissions: Don't make files world-writable
  • Never trust user input: Validate and sanitize filenames

Secure Upload Example:

<?php
  if(isset($_FILES["file"])) {
    // Generate unique filename
    $extension = pathinfo($_FILES["file"]["name"], PATHINFO_EXTENSION);
    $uniqueName = uniqid() . "_" . time() . "." . $extension;
    
    // Validate file type
    $allowedTypes = ["image/jpeg", "image/png", "image/gif"];
    $fileType = $_FILES["file"]["type"];
    
    if(in_array($fileType, $allowedTypes)) {
      // Validate file size (max 2MB)
      if($_FILES["file"]["size"] <= 2097152) {
        $targetPath = "uploads/" . $uniqueName;
        
        if(move_uploaded_file($_FILES["file"]["tmp_name"], $targetPath)) {
          echo "File uploaded securely as: " . $uniqueName;
        }
      } else {
        echo "File too large (max 2MB)";
      }
    } else {
      echo "Invalid file type";
    }
  }
?>
